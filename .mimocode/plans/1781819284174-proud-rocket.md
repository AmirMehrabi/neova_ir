# Neova Dashboard Redesign — Options

## Problem Statement
- Dashboard (`/dashboard`) and Workspace page (`/workspace`) look identical
- No useful statistics — just flat card lists
- 3 clicks minimum to reach a board: Login → Dashboard → Workspace → Board
- Header says "تخته اسکرام" instead of "نئووا"

## Current State
- Both pages: identical blue header → title + button → 3-col card grid → modal
- Dashboard shows: workspace name, project count, owner
- Workspace shows: project name, key, column count, task dots
- Data available: `$workspaces` (with `projects`), `$projects` (with `columns`)
- Route structure: `/{workspace}` → `/{workspace}/{project}/board`

---

## Option A: Single Unified Page (Merge)

**Concept:** Eliminate the workspace page entirely. Dashboard becomes a single page with a sidebar for workspace selection and main area for project cards + stats.

```
│  Header: [N نئووا]              [Avatar] [خروج] │

│ Sidebar  │  │ ۳   │ │ ۱۲  │ │ ۸   │        │
│          │  │پروژه│ │وظیفه│ │انجام│        │
│ فضای کاری│  └─────┘ └─────┘ └─────┘        │
│ ────────│──────────────────────────────────│
│ ▶ من     │  پروژه‌ها                          │
│   تیم    │  ┌────────┐ ┌────────┐          │
│          │  │ NEO-001│ │ NEO-002│          │
│ + جدید   │  │ پس‌زمینه│ │ در حال │          │
│          │  │ انجام  │ │ انجام  │          │
└──────────┴──────────────────────────────────┘
```

**Clicks to board:** 2 (select workspace in sidebar → click project)
- Auto-selects user's only workspace on load
- If user has 1 workspace: 1 click to board

**Changes:**
| File | Action |
|------|--------|
| `resources/views/dashboard.blade.php` | **Rewrite** — unified layout with sidebar |
| `resources/views/workspace.blade.php` | **Delete** |
| `app/Http/Controllers/DashboardController.php` | **Update** — single index with `?workspace=slug` param |
| `routes/web.php` | **Update** — remove `/{workspace}` route |

**Pros:** Simplest navigation, one page to maintain, fewer routes
**Cons:** Losing dedicated workspace management page (settings, members, etc. future feature)

## Recommendation

**Option B** (Distinct Pages + Quick Access) best addresses all three goals:
1. **Visual differentiation** — completely different layouts (home vs management)
2. **Statistics** — dashboard has cross-workspace stats, workspace has per-project stats
3. **Minimum clicks** — quick access section provides 1-click board navigation

Option A is simpler but loses the dedicated workspace page. Option C is the least disruptive but doesn't fully solve the confusion.

- Logo: Dark navy square with white "N"
