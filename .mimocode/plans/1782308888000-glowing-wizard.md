# Fix Priority vs Column Color Conflict

## Problem
Priority and column colors overlap:
- `بررسی` (Review) column = `#F59E0B` amber = `متوسط` (Medium) priority = `#F59E0B` amber
- `انجام شده` (Done) column = `#22C55E` green = `پایین` (Low) priority = `#22C55E` green

## Solution
Keep column colors unchanged (they're more prominent). Change priority colors to a distinct palette that doesn't clash.

### New Priority Color Palette

| Priority | Current | New | Reason |
|----------|---------|-----|--------|
| بالا (High) | `#EF4444` red | `#DC2626` red (keep) | Red is universal for high urgency, no column uses it |
| متوسط (Medium) | `#F59E0B` amber | `#8B5CF6` purple | Purple is distinct from all column colors |
| پایین (Low) | `#22C55E` green | `#64748B` slate gray | Neutral tone, clearly different from Done's green |

### Visual Result
- High: Red bar (urgent, attention-grabbing)
- Medium: Purple bar (distinct, moderate)
- Low: Slate bar (subdued, calm)

None of these conflict with column colors (gray, blue, amber, green).
