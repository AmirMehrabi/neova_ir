# Workspace Invitation Banner on Dashboard

## Goal
Add a prominent invitation banner at the top of the dashboard that displays pending workspace invitations with accept/decline buttons, making them more visible than the notification menu.

## Implementation

### 1. Update DashboardController
**File:** `app/Http/Controllers/DashboardController.php`

Add query to fetch pending invitations for the current user:
```php
$invitations = \App\Models\WorkspaceInvitation::query()
    ->with(['workspace', 'inviter'])
    ->where('phone', $user->phone)
    ->where('status', 'pending')
    ->where('expires_at', '>', now())
    ->get()
    ->each->markExpiredIfNeeded()
    ->filter->isPending();
```

Pass `$invitations` to the view.

### 2. Add Invitation Banner to Dashboard View
**File:** `resources/views/dashboard.blade.php`

Insert a new section between the header (line 191) and the workspaces container (line 193). This section will:

- Only render if there are pending invitations
- Display stacked cards for each invitation
- Each card shows:
  - Inviter's avatar (initials)
  - Inviter's name
  - Workspace name being invited to
  - Role being offered
  - Accept button (POST to `invitations.accept`)
  - Decline button (POST to `invitations.decline`)
- Use Alpine.js for potential animations/transitions
- Follow existing design system (colors, spacing, rounded corners)

### 3. Create Invitations Component (Optional)
**File:** `resources/views/components/invitation-banner.blade.php`

Extract the invitation banner into a reusable component for cleaner code.
