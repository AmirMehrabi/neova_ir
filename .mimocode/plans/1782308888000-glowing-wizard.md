# Project Visibility System

## Goal
Introduce project-level visibility (Public/Private) so workspace owners and admins can control which projects are visible to all members vs. only project team members.

## Visibility Levels
- **Public** (`public`): Visible to all workspace members (current behavior)
- **Private** (`private`): Visible only to project members + workspace owner + workspace admins

Default for new projects: **Public**

---

## Implementation Steps

### Step 1: Database Migration
**New file:** `database/migrations/2026_06_24_000001_add_visibility_to_projects_table.php`

Add a `visibility` column to the `projects` table:
- Column: `visibility`, string(20), default `'public'`, not nullable
- Values: `'public'` or `'private'`

```php
Schema::table('projects', function (Blueprint $table) {
    $table->string('visibility', 20)->default('public')->after('key');
});
```

### Step 2: Project Model Changes
**File:** `app/Models/Project.php`

1. Add `'visibility'` to `$fillable` array
2. Add `canUserView(User $user, Workspace $workspace): bool` method:
   - If `visibility === 'public'`: return true (all workspace members can see)
   - If `visibility === 'private'`:
     - Return true if user is workspace owner (`$workspace->isOwnedBy($user)`)
     - Return true if user has workspace role `admin` (`$workspace->roleFor($user) === 'admin'`)
     - Return true if user is a project member (`$this->members()->where('user_id', $user->id)->exists()`)
     - Otherwise return false

### Step 3: New Middleware — EnsureProjectAccess
**New file:** `app/Http/Middleware/EnsureProjectAccess.php`

This middleware runs **after** `EnsureProject` (which resolves the project). It checks:
1. Get `$project` and `$workspace` from request attributes
2. Call `$project->canUserView($request->user(), $workspace)`
3. If false → abort(403)

Register in `bootstrap/app.php`:
```php
'project.access' => \App\Http\Middleware\EnsureProjectAccess::class,
```

### Step 5: Dashboard Filtering
**File:** `app/Http/Controllers/DashboardController.php`

In `index()`, after loading workspaces with projects, filter each workspace's projects:
```php
$workspace->setRelation('projects', $workspace->projects->filter(
    fn ($project) => $project->canUserView($user, $workspace)

```

Also update `withCount('projects')` to use a scoped query that respects visibility (for accurate counts). This requires a subquery approach:
```php
->withCount(['projects as projects_count' => function ($q) use ($user, $workspaceIds) {
    $q->where(function ($sub) use ($user) {
        $sub->where('visibility', 'public')
            ->orWhere(fn ($s) => $s->where('visibility', 'private')
                ->where(fn ($ss) => $ss->where('workspace_id', $user->ownedWorkspaces()->select('id'))
                    ->orWhere(fn ($sss) => $sss->where('workspace_id', $user->workspaces()->select('id'))
                        ->where('workspace_members.role', 'admin'))));
    });
}])
```

Actually, simpler approach: do visibility filtering in-memory after fetching, and adjust the project count manually. The eager-loaded collection filter is cleaner.

**Revised approach:** Filter projects in the `each` callback and update `projects_count` attribute:
```php
->each(function (Workspace $workspace) use ($user) {
    $workspace->setAttribute('user_role', $workspace->roleFor($user));
    
    $visibleProjects = $workspace->projects->filter(
        fn ($project) => $project->canUserView($user, $workspace)
    )->values();
    $workspace->setRelation('projects', $visibleProjects);
    $workspace->setAttribute('projects_count', $visibleProjects->count());
    
    // ... existing progress calculation
});
```

### Step 6: Search Filtering
**File:** `app/Http/Controllers/DashboardController.php`

In `search()`, filter projects and tasks to only include visible ones:

For projects query, add visibility filter:
```php
$projects = Project::whereIn('workspace_id', $workspaceIds)
    ->where('name', 'LIKE', "%{$query}%")
    ->where(function ($q) use ($user) {
        $q->where('visibility', 'public')
          ->orWhere(fn ($s) => $s->where('visibility', 'private')
              ->where(function ($ss) use ($user) {
                  $ss->where(fn ($sss) => $sss->whereIn('workspace_id', $user->ownedWorkspaces()->select('id')))
                    ->orWhere(fn ($sss) => $sss->whereIn('workspace_id', $user->workspaces()->wherePivot('role', 'admin')->select('id')))
                    ->orWhere(fn ($sss) => $sss->whereIn('id', DB::table('project_members')->where('user_id', $user->id)->select('project_id')));
              }));
    })
    ->with('workspace')
    ->limit(5)
    ->get();
```

For tasks query, filter by project visibility similarly.

### Step 9: Controller Method for Visibility Toggle
**File:** `app/Http/Controllers/WorkspaceManagementController.php`

Add `updateProjectVisibility()` method:
```php
public function updateProjectVisibility(Request $request, string $workspaceSlug, string $projectSlug)
{
    $workspace = $this->workspace($request, $workspaceSlug);
    abort_unless($workspace->canManageMembers($request->user()), 403);

    $project = $workspace->projects()->where('slug', $projectSlug)->firstOrFail();

    $validated = $request->validate([
        'visibility' => 'required|in:public,private',
    ]);

    $project->update(['visibility' => $validated['visibility']]);

    return back()->with('success', 'visibility_updated');
}
```
