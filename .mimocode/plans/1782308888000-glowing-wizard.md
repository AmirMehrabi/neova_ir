# Board Features: Activity Tab, Task Filtering & Search

## Goal
Add three features to the board page:
1. **Activity tab** in the project management drawer with search
2. **Filter tasks by assignee** — toggle to show only tasks assigned to specific people
3. **Search with highlight** — search bar like dashboard's, filters and highlights matching tasks

---

## Feature 1: Activity Tab

### What
A new "فعالیت‌ها" (Activity) tab in the project management drawer showing all task activity (creates, updates, moves, mentions, comments) with search.

### Data Source
Query `notifications` table where `data->project_id` matches the current project, ordered by latest.

### Controller Changes
**File:** `app/Http/Controllers/BoardController.php`

Add a new route for activity:
```php
Route::get('/{workspace}/{project}/activity', [BoardController::class, 'activity'])->name('board.activity');
```

**New method:** `activity(Request $request)`
- Query notifications for the project
- Paginate or limit to 50
- Return JSON with activity items

### Alpine.js State (in board.blade.php)
```
activityTab: 'members',    // 'members' | 'settings' | 'activity'
activitySearch: '',
activityItems: [],
activityLoading: false,
```

### New Methods
- `loadActivity()` — fetches activity from server, stores in `activityItems`
- `filteredActivity()` — filters `activityItems` by `activitySearch`
- `highlightText(text, query)` — returns HTML with `<mark>` tags around matches

### UI
Third tab in drawer: "فعالیت‌ها". Shows a search input + list of activity items. Each item shows:
- Icon (create/update/move/mention)
- Actor name + action description (from `data.message`)
- Timestamp
- Search matches highlighted in `<mark>` tags

|------|--------|

| `routes/web.php` | Add activity route |
| `resources/views/board.blade.php` | Add tab button, activity section, Alpine methods |

## Feature 2: Filter Tasks by Assignee

### What
A filter bar above the board columns showing project member avatars. Clicking a member toggles filtering to show only their tasks across all columns. Multiple members can be selected (OR logic). A "clear" button resets.

### Alpine.js State
```
filterByAssignee: [],     // array of member names to filter by
```

### Alpine.js Methods
- `toggleAssigneeFilter(name)` — adds/removes name from `filterByAssignee`
- `isAssigneeFiltered(task)` — returns true if task should be visible (no filter active OR task has at least one matching assignee)
- `clearAssigneeFilter()` — empties `filterByAssignee`
- `filteredTasks(column)` — returns column.tasks filtered by `isAssigneeFiltered()`

### Task Rendering Change
Both desktop and mobile task `x-for` loops change from:
```html
x-for="task in column.tasks"
```
to:
```html
x-for="task in filteredTasks(column)"
```

### UI
A horizontal bar below the board header showing:
- "فیلتر:" label
- Each project member as a small avatar pill (circle + name)
- Active filters highlighted in blue
- Clear button when any filter is active

### Files to Modify
| File | Change |
|------|--------|
| `resources/views/board.blade.php` | Add filter bar HTML + Alpine methods + modify task x-for loops |

## Feature 3: Search with Highlight

### What
A search bar in the board header (similar to dashboard's) that filters tasks across all columns and highlights matching text in task titles and descriptions.

### Alpine.js State
```
boardSearchQuery: '',
boardSearchOpen: false,
```

### Alpine.js Methods
- `filteredTasks(column)` — enhanced to also check `boardSearchQuery` against task title + description
- `highlightText(text, query)` — wraps matches in `<mark style="background:#FEF3C7;padding:0 2px;border-radius:3px">` tags
- `boardSearchResults()` — returns count of matching tasks across all columns

### Task Rendering Change
The `filteredTasks(column)` method combines both assignee filter AND search query:
```js
filteredTasks(column) {
    return column.tasks.filter(task => {
        const matchesAssignee = this.filterByAssignee.length === 0 ||
            task.assignees.some(a => this.filterByAssignee.includes(a));
        const matchesSearch = !this.boardSearchQuery ||
            task.title.toLowerCase().includes(this.boardSearchQuery.toLowerCase()) ||
            task.description.toLowerCase().includes(this.boardSearchQuery.toLowerCase());
        return matchesAssignee && matchesSearch;
    });
}
```

### UI
Search bar in the board navbar (dark header), similar to dashboard:
- Icon + input + dropdown results
- As user types, tasks across all columns are filtered in real-time
- Matching text in task cards gets `<mark>` highlight
- Shows result count: "۳ نتیجه یافت شد"
- Press Escape or clear to reset

### Task Card Changes
Title and description rendering change from:
```html
<p x-text="task.title"></p>
```
to:
```html
<p x-html="highlightText(task.title, boardSearchQuery)"></p>
```
