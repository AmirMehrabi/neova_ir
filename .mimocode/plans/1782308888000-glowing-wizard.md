# Email Notification System

## Goal
Build a robust, Basecamp-style email notification system that feels personal and dynamic. Emails should be in Farsi with the Neova logo, using warm and human language.

---

## Email Types & Examples

### 1. Task Assignment Email
**Subject:** {{actor}} شما را به «{{task}}» اضافه کرد

**Preview:** در پروژه {{project}}

**Body (Farsi):**
```
سلام {{user.first_name}},

{{actor}} شما را به وظیفه «{{task}}» در پروژه «{{project}}» اضافه کرد.

[دیدن وظیفه →]
```

## Implementation Steps

### Step 1: Email Layout Template
**New file:** `resources/views/emails/layout.blade.php`

A shared HTML email layout with:
- Neova logo (horizontal, dark version)
- RTL direction, Farsi typography
- Warm background (#f4f0e7)
- Clean, readable design
- Footer with unsubscribe/settings link

### Step 2: Email View Templates
**New files in `resources/views/emails/`:**
- `task-assigned.blade.php`
- `task-updated.blade.php`
- `task-moved.blade.php`
- `task-mentioned.blade.php`
- `workspace-invitation.blade.php`
- `project-updated.blade.php`
- `daily-digest.blade.php`

### Step 3: Update Notification Classes
**Files:** `app/Notifications/ProjectActivityNotification.php`, `app/Notifications/WorkspaceInvitationNotification.php`

Add `toMail()` method and include `'mail'` in `via()` when:
- User has email address set
- User has the relevant notification preference enabled

### Step 4: Update Notification Services
**Files:** `app/Services/ProjectActivityNotifier.php`, `app/Services/WorkspaceInvitationService.php`

Add preference checks before sending:
- Check `hasNotificationPreference('task_activity')` for task notifications
- Check `hasNotificationPreference('invitations')` for invitation notifications
- Check `hasNotificationPreference('project_updates')` for project notifications

### Step 5: Daily Digest Job
**New file:** `app/Jobs/SendDailyDigestJob.php`

- Collects all task activity from the past 24 hours for each user
- Groups by project
- Sends a single digest email if user has `digest` preference enabled
- Scheduled via `routes/console.php` (daily at 8 AM)

### Step 6: Update .env
Set `APP_NAME=Neova` and update `MAIL_FROM_ADDRESS` to a proper address.

## Verification

1. **Test task email:** Assign a task → verify email logged (or sent if SMTP configured)
2. **Test invitation email:** Invite user → verify invitation email sent
3. **Test preference check:** Disable task_activity preference → verify no email sent
4. **Test digest:** Run `php artisan digest:send` → verify digest email content
5. **Test email layout:** Verify logo displays, RTL works, Farsi renders correctly
