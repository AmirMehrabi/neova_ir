# User Profile, Avatar & Notification Preferences

## Goal
1. Fix navbar avatar to show uploaded image or fallback to initials
2. Create a comprehensive /profile page with avatar upload, email, and notification preferences
3. Add email field to registration (optional)
4. Enable notification preference management

---

## Implementation Steps

### Step 1: Database Migration
**New file:** `database/migrations/2026_06_24_000002_add_email_and_notification_prefs_to_users_table.php`

Add columns to `users` table:
- `email` — string, nullable, unique (for notification purposes)
- `notification_preferences` — json, default: all enabled

```php
Schema::table('users', function (Blueprint $table) {
    $table->string('email')->nullable()->unique()->after('phone');
    $table->json('notification_preferences')->nullable()->after('avatar');
});
```

Default notification preferences structure:
```json
{
    "task_activity": true,
    "invitations": true,
    "project_updates": true,
    "digest": false
}
```

### Step 2: User Model Updates
**File:** `app/Models/User.php`

1. Add `email` and `notification_preferences` to fillable
2. Add cast for `notification_preferences` as `array`
3. Add `getAvatarUrlAttribute()` accessor:
   - If `avatar` is set: return the storage URL
   - Otherwise: return null (fallback to initials in views)
4. Add helper methods:
   - `getInitialsAttribute()` — returns first letter of first_name or name
   - `hasNotificationPreference(string $key): bool`

### Step 3: Profile Controller
**New file:** `app/Http/Controllers/ProfileController.php`

Methods:
- `show()` — render profile page with user data
- `update(Request)` — update profile fields (first_name, last_name, email)
- `updateAvatar(Request)` — handle avatar image upload
- `removeAvatar(Request)` — delete avatar, revert to initials
- `updateNotificationPreferences(Request)` — update notification settings

Avatar upload logic:
- Validate: image, max 2MB, mimes: jpg,jpeg,png,webp
- Store in `storage/app/public/avatars/{user_id}.ext`
- Delete old avatar if exists
- Update user's `avatar` field with filename

### Step 4: Profile Routes
**File:** `routes/web.php`

Add authenticated routes:
```php
Route::get('/profile', [ProfileController::class, 'show'])->name('profile');
Route::patch('/profile', [ProfileController::class, 'update'])->name('profile.update');
Route::post('/profile/avatar', [ProfileController::class, 'updateAvatar'])->name('profile.avatar');
Route::delete('/profile/avatar', [ProfileController::class, 'removeAvatar'])->name('profile.avatar.destroy');
Route::patch('/profile/notifications', [ProfileController::class, 'updateNotificationPreferences'])->name('profile.notifications');
```

### Step 5: Profile View
**New file:** `resources/views/profile.blade.php`

Sections:
1. **Avatar Section**
   - Current avatar display (image or initials fallback)
   - Upload button with file input
   - Remove avatar button (if avatar exists)
   
2. **Personal Information**
   - First name (editable)
   - Last name (editable)
   - Phone (display only, not editable)
   - Email (editable, for notifications)
   
3. **Notification Preferences**
   - Toggle switches for each preference:
     - Task activity emails (assignments, comments, mentions)
     - Invitation emails
     - Project update emails
     - Digest emails (daily/weekly summary)
