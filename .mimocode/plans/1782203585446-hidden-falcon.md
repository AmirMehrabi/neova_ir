# Fix: Remove "001" prefix from task names

## Root Cause

`BoardController::storeTask()` (line 119) embeds an auto-generated ID directly into the `title` string:

```php
'title' => $project->key.'-'.str_pad($maxNum + 1, 3, '0', STR_PAD_LEFT).' '.$request->title,
```

This produces titles like `-001 ریدیزاین...` (note: project key is empty because the project name is Persian and the key generator strips non-ASCII chars). The number is calculated by regex-parsing existing titles from the DB, which is fragile. The user sees the "001" prefix baked into every task name and wants it removed.

## Approach

Separate the task number from the title. Add a `task_number` column, store only the user-entered title, and compute the display ID (`{project_key}-{task_number}`) on the fly.

## Changes

### 1. Migration: Add `task_number` column
**File**: `database/migrations/2026_06_23_000001_add_task_number_to_tasks_table.php` (new)

- Add nullable `integer` column `task_number` to `tasks` table
- Backfill existing tasks: parse current titles to extract the number, then strip the prefix from the title

### 2. Task Model
**File**: `app/Models/Task.php`

- Add `task_number` to `$fillable`
- Add accessor `display_id` that computes `{project_key}-{str_pad(task_number, 3, '0')}` from the parent project

### 3. BoardController::storeTask()
**File**: `app/Http/Controllers/BoardController.php` (lines 106-126)

- Compute `task_number` from existing tasks (same regex logic, but store as separate field)
- Store `task_number` in DB
- Store only the user-entered title (no prefix)
- Return `display_id` in JSON response

### 4. BoardController::show()
**File**: `app/Http/Controllers/BoardController.php` (lines 43-60)

- Compute `display_id` for each task: `{project.key}-{str_pad(task_number, 3, '0')}`
- Pass as a separate field in `columnsData` JSON (alongside `id`/`dbId`/`title`)

### 5. Board frontend
**File**: `resources/views/board.blade.php`

- Card badge (line 173): show `task.displayId` (e.g., "SCR-001") instead of `task.id` (full title)
- Card title (line 180): show `task.title` (clean, no prefix) — already correct after backend change
- Modal header badge (line 495): show `form.displayId`
- `openEditModal` (line 1516): populate `form.displayId` from `task.displayId`; `form.title` stays clean
- `openAddModal` (line 1505): `form.displayId = ''` (not yet assigned)
- `saveTask` create response (line 1559): capture `display_id` from response and set on task object

### 6. Backfill migration logic
- Parse each existing title: extract number via `/^{key}-(\d+)/` or `^-(\d+)/` fallback
- Update `task_number` column
- Strip the prefix from `title` (remove everything up to and including the first space after the number)
